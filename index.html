New ECMAScript features

var, let, const

var vs. let

ES 5

                /* global scope, function scope */
                var a = 5;
ES 6

                /* block scope */
                let a = 5

example - var


              var elements = ['A', 'B', 'C'];
      
              for (var i = 0; i < elements.length; i++) {
                console.log('element: ', elements[i]);
              }

              console.log(i); /* will print 3 */

example - let


              var elements = ['A', 'B', 'C'];
      
              for (let i = 0; i < elements.length; i++) {
                console.log('element: ', elements[i]);
              }

              console.log(i);
              /* Uncaught ReferenceError: i is not defined */

example 3 - const object

Functional programming

Imperative vs. declarative programming

Imperative code


              const getTotalFortuneOfTenRichest = richPeople => {
                richPeople.sort((first, second) => {
                  return first.money - second.money;
                });

                let sum = 0;
            
                for (let i = 0; i < richPeople.length; i++) {
                    sum += richPeople[i].money;
            
                    if (i >= 10) {
                        return sum;
                    }
                }
              };

Imperative code

  • sort function mutates the array
  • sum variable is mutable
  • code is hard to read - we declare the execution flow (it looks like alghoritm)

Declarative code (not fully)


              const getTotalFortuneOfTenRichest = richPeople => {
                return [...richPeople]
                  .sort((first, second) => first.money - second.money)
                  .slice(0, 10)
                  .reduce((total, person) => total + person.money, 0)
              };

Declarative code (not fully)

  • [...richPeople] is copy of richPeople - no mutation
  • no redundant variables like sum
  • code is easier to read - we define what we expect rather than execution steps

Pure functions

Pure function - example 1


              const double = number => {
                return number * 2;
              };

Pure function - example 2


              const factorial = (n) => {
                if (n === 0) {
                  return 1;
                }
            
                /* if n=5, it will create an array: [1,2,3,4,5] */
                const arr = Array(n).fill(n).map(
                  (num, index) => num - index
                );

                /* for n=5, 5 * 4 * 3 * 2 * 1 */
                return arr.reduce((acc, num) => acc * num, 1);
              }
or 

              return [...arr].reverse().reduce(
                (acc, num) => acc * num,
                arr[0]
              );

Pure function - definition

  • Given the same input, will always return the same output.
  • Produces no side effects - it does NOT mutate any data.

React fundamentals

What is React?

JavaScript library for building user interfaces
(created by Facebook)
  • It is NOT a framework
  • It is NOT opinionated (unlike Angular)
  • It is component-based
  • It uses custom syntax - JSX
  • It has no DSL* (unlike Angular)
  • It uses Virtual DOM
(*) Domain-specific Language

Virtual DOM

  • Data structure, which represents real DOM
  • During change, React compares changes on this structure and knows what part of DOM should be updated

JSX

(JavaScript XML -> JS + HTML combined)

example 1


              const name = 'Joe Doe';
              const element = <h1>Hello, {name}</h1>

              ReactDOM.render(
                element,
                document.getElementById('root')
              );

example 2


              const formatName = ({firstName, lastName}) => {
                return `${firstName} ${lastName}`
              };
              
              const user = {
                firstName: 'Joe',
                lastName: 'Doe'
              };
              
              const element = (
                <h1>
                  Hello, {formatName(user)}!
                </h1>
              );
              
              ReactDOM.render(
                element,
                document.getElementById('root')
              );

Components

Components

  • Pure functions, which contains JSX
  • Should be named PascalCase (React cannot confuse components with HTML tags)
  • Has lifecycle
  • Has state
  • Has props

Class component


              class MyComponent extends React.Component {
                name = 'React'

                render() {
                  return (
                    <div>
                      Welcome {this.name}!
                    </div>
                  )
                }
              }

Function component


              const MyComponent = () => {
                const name = 'React'

                return (
                  <div>
                    Welcome {name}!
                  </div>
                )
              }
